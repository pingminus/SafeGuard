# Contributing

Thanks for your interest in contributing to this project!

## How to contribute
1. Fork the repository
2. Create a new branch
3. Make your changes
4. Commit your changes with a clear message
5. Push your branch to your fork
6. Open a Pull Request

## Code style
- Follow the existing code style
- Keep code readable and well-documented
- Avoid unnecessary dependencies

## Features & Improvements
Contributions are very welcome, including but not limited to:
- New features (especially quality-of-life improvements)
- Fixes for readability, grammar, or spelling
- Adding new encryption algorithms (for educational purposes)
- Refactoring or performance improvements
- Documentation improvements

If you’re unsure whether a change fits the project, feel free to open an issue or pull request and discuss it.
